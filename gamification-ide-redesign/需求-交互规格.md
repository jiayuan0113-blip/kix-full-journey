## 0. 一句话目标
商家进入工作台 5 秒内明白「把这个游戏改成我的品牌 → 试玩 → 发布」，全程能**不打字**地把屏上的字/图换成自己的，改完当场能玩。

## 1. 信息架构（一屏三栏 + 顶栏）

```
┌──────────────────────────────────────────────────────────────┐
│ 顶栏：KiX Logo · 游戏名 + 副标(改成你的品牌→试玩→发布)          │
│        ……………………………  撤销/重做 · 中|EN ·  [头像(最右)]        │
├───────────────┬──────────────────────────┬───────────────────┤
│ 左：AI 助手    │ 中：预览区（主角）        │ 右：素材（上传图） │
│ 常驻对话       │  [存草稿] [发布]（居中）  │ 分类平铺：         │
│ + 默认推荐卡   │  状态标签(编辑中/试玩中)  │  Logo / 背景 /     │
│ + 输入框       │  📱 9:16 手机预览          │  商品图            │
│               │  [试玩 ⇄ 编辑] 单一切换    │                   │
└───────────────┴──────────────────────────┴───────────────────┘
```

- **顶栏**：左=Logo + 单一游戏名（可点改名）+ 一行副标「改成你的品牌 → 试玩 → 发布」；右=撤销/重做、语言(中|EN)、**头像固定最右角**。
- **中列（唯一主角）**：预览手机 + 其正上方的 `存草稿/发布` + 状态标签 + 其正下方的 `试玩/编辑` 切换。
- **左列**：AI 助手，常驻（非弹窗）。
- **右列**：素材面板，只管**上传各类图**，按分类平铺。

## 2. 核心：编辑态 ⇄ 试玩态 状态机（最重要）

预览里跑的是真游戏（match-3，会倒计时、方块会动）。游戏运行时无法稳定点选元素去改；冻结时又体验不到节奏。故二者**互斥**，必须显式切换。

| | **编辑态（默认 / home）** | **试玩态** |
|---|---|---|
| 游戏 | **冻结在初始帧**，不倒计时、不自动消除 | **真实运行**：倒计时、可玩、可消除加分 |
| 元素 | 文字/图**可点即改**（见 §4.2） | 全部不可编辑（防误触） |
| 编辑 chrome | 显示（可编辑元素有可点态） | 隐藏，呈现「客人看到的样子」 |
| 状态标签 | 「编辑中」（蓝点） | 「试玩中」（绿点） |
| 切换按钮文案 | 显示「试玩」 | 显示「编辑」 |

**规则**
1. 进入工作台默认 **编辑态**。
2. 预览正下方**单一切换按钮**在两态间翻转（编辑态=「试玩」；试玩态=「编辑」）。当前态由状态标签 + 是否有编辑外框双重可见。
3. **没有逐元素「保存/确认」**：编辑态里改动**实时生效**到冻结预览；持久化只由顶部 `存草稿 / 发布` 承担。
4. 试玩态倒计时归零 → 结算（弹分数）→ 自动回编辑态。
5. 试玩态下点击任何素材/上传入口 → 提示「先结束试玩」。
6.（可选增强，需 A/B）进入时先自动试玩 2–3s 展示「这是个活游戏」，随即停回编辑态。

## 4. 中列（预览区）

### 4.1 存草稿 / 发布（位于预览正上方、居中）
- `存草稿`：次级按钮，暂存草稿。
- `发布`：主色按钮，主 CTA。**发布是硬闸门**：未登录先要手机号验证码（注册后置到发布，登录=注册同一动作）；发布走审批（活动状态机 draft→review→live，见后端 canon）。
- 二者与手机同处中轴，强调「改的就是这台手机、发布的也是它」。

### 4.2 手机预览（9:16）— 编辑态就地改
- **改文字**：编辑态下点「游戏标题」「玩法说明」等文字 → 就地光标输入 → 点空白/回车即生效（无独立确认）。
- **点商品方块**：编辑态下点方块 → 高亮右栏「商品图」分类做引导（**不弹模态**），提示去右侧换/上传。
- 试玩态下以上均不可编辑。

## 5. 左列：AI 助手（常驻，非弹窗）

- **常驻侧栏**，进入即见输入框，不需点开（删除旧版右下角浮动入口）。
- **默认推荐卡**（可点即执行，无需打字）：
  1. 换品牌配色和 Logo　2. 换成我的商品图　3. 节奏调快一点，更上瘾　4. 帮我改标题文案
- 卡片点击直接产生可见改动（换色/换商品/调时长/聚焦标题）+ 一条 AI 回复气泡。
- 输入框 placeholder：「不知道改什么？描述一下，或从上面挑一个」。
- 进阶（建议）：点预览里的元素 → AI 输入自动带上「针对这个元素」的上下文。

## 6. 右列：素材（上传图，按分类平铺，无弹窗）

右栏**只管上传/切换图**，不含文案（文案在预览上改）。三个分类平铺，缩略图直接可点，**当前生效项高亮打勾**：

| 分类 | 内容 | 上传后自动处理 |
|---|---|---|
| **Logo** | 「＋上传」+ 已传的 Logo | 上传即**自动取品牌主色**（驱动全局 accent） |
| **背景** | ①场景图（平台示例）②纯色/渐变 ③「＋上传」整图 | 上传整图**自动裁切适配 9:16** |
| **商品图** | 商品图套（示例）+ 已传的 + 「＋上传」 | 上传商品照**自动裁成方块图标** |

**交互规则**
1. 点任一缩略图 = **就地套用**，实时改预览，**不弹模态**。
2. 「＋上传」= 分类型上传（从哪个分类进就是哪类图），上传的图进本分类那一行、**可复用**。
3. 背景支持三来源并列：场景图 / 纯色·渐变（干净品牌底色）/ 上传整图。
4. 「这张图能用到哪」永远由**它所在的分类**决定，不会出现「传了不知道放哪」。

## 7. 现状穿帮（P0，必修，先于重排）

来自线上 partner-test 评审，属上线/演示阻断项：
- **P0-1 预览漏中文**：EN 模式下计时器显示中文档位词（`慢悠悠/慢慢来`）。修：计时器 label 用图标或 locale 串；中/EN 切换必须同时切**预览内运行时文案**。
- **P0-2 中文源码注释进 bundle**：对外环境强制 minify + strip comments。
- **P0-3 `Deep` 占位标签**：chip 上的 `Deep` pill 去掉或换成可懂状态词。
- **P1 chip 死值**：`Coffee Items` 等副值改成反映当前游戏真实态。

## 8. 数据 / 素材模型（给后端对齐）

- **游戏(Game)** = 纯视觉：玩法模板 + 品牌视觉（Logo/主色/背景/商品图）+ 文案。本工作台改的就是 Game 的视觉层。
- **可品牌化字段**：`title`(文案)、`accent/logo`、`background`(场景 key | 颜色值 | 上传图)、`products[]`(图标集 | 上传图集)、`round_time`(节奏)。
- **素材库**：商家上传的图按 `type ∈ {logo,bg,product}` 归类、可跨活动复用。
- **语言**：外壳 + 运行时文案都要过 i18n；locale 切换驱动预览内文本。
- 发布闸门与活动状态机（draft/review/live）沿用后端既有 canon。
