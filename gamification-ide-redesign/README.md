# Gamification IDE · 重新设计（交互原型 + 需求）

商家用 AI 建品牌小游戏的**建游戏工作台**（Gamification IDE）重设计交付包。给研发对照实现。

## 目录
| 文件 | 说明 |
|---|---|
| `KiX-IDE-Redesign-Demo.html` | 可点交互原型（**双击即开**，离线可跑）。URL 加 `#play` 直接进试玩态。 |
| `需求-交互规格.md` | **研发必读**：信息架构、编辑⇄试玩状态机、三栏交互、素材模型、P0 必修项。 |
| `设计方案.md` | 设计决策与病根分析（为什么这么改）。 |
| `现状评审.md` | 线上 partner-test 现状问题清单（P0/P1 穿帮）。 |

## 一句话
让商家进来 5 秒内明白「把这个游戏改成我的品牌 → 试玩 → 发布」，全程不打字就能把屏上的字/图换成自己的，改完当场能玩。

## 核心交互
- **编辑态 ⇄ 试玩态**：预览手机是唯一主角；编辑态冻结可改、试玩态运行不可改，单一按钮切换，无逐元素保存。
- **左 AI 常驻 + 默认推荐**：不弹窗，进来即用。
- **右素材按分类平铺**：Logo / 背景 / 商品图，点即用、分类型上传自动适配（背景 9:16 / 商品裁图标 / Logo 取色），不弹窗。
- **文案在手机上就地改**；`存草稿 / 发布` 居中于预览区，发布为硬闸门。

> 原型用 emoji 占位真实图片，仅演示交互与信息架构，不代表最终视觉素材。
